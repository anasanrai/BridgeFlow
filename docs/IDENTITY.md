- Honest even when it's uncomfortable

### What We Don't Sound Like
- Corporate jargon ("synergize", "leverage", "holistic")
- Fake enthusiasm ("We're SO excited to partner with you!")
- Vague promises ("Transform your business")
- Over-technical (talking to engineers when clients are ops managers)
- Overly humble (we're good at what we do — say it)

### Tone by Context

| Context | Tone | Example |
|---|---|---|
| **Website copy** | Confident, punchy, direct. Short sentences. Benefits over features. Numbers wherever possible. | "3 workflows. 10 integrations. Live in 2 weeks." |
| **Email** | Warm but professional. No filler phrases. Get to the point in sentence 1. | "Your workflow is live. Here's the walkthrough video." |
| **Social media** | Behind the scenes, honest, educational. Show the work. Show the failures. Show the growth. | "This automation took 4 hours to build. It saves the client 12 hours every week." |
| **Client communication** | Clear, fast, proactive. Never make the client ask for an update. | "Day 3 update: CRM integration is done. Email sequence starts tomorrow. On track for Friday delivery." |
| **Sales calls** | Consultative, listening-first, zero pressure. Ask about their pain before talking about our solution. | "Walk me through what happens when a new lead comes in right now." |
| **Proposals** | Structured, specific, no ambiguity. Scope, timeline, price — nothing left to interpretation. | "Scope: 3 workflows. Timeline: 14 business days. Price: $997. Payment: 50/50." |

### Words We Use

### Logo

**Construction:**
- **Logomark:** Stylized "B" formed by two connected bridge arcs,
  suggesting data flowing across a bridge. Clean geometric construction.
  Works at 16px favicon size and 200px+ hero size.
- **Wordmark:** "BridgeFlow" in Inter Bold (or custom lettering),
  tight tracking (-0.02em), with "Bridge" in text primary color
  and "Flow" in coral red (#ff6d5a).
- **Lockup:** Logomark left, wordmark right. 8px gap at standard size.

**Usage rules:**
- Never stretch, skew, or rotate
- Never recolor outside approved palette
- Minimum clear space: 1x the height of the logomark on all sides
- Minimum size: 24px height for digital, 10mm for print
- On dark backgrounds: white wordmark, coral logomark
- On light backgrounds: dark wordmark, coral logomark
- Never place on busy/photographic backgrounds without a solid backing

### Illustration & Imagery Style

| Type | Direction |
|---|---|
| **Diagrams** | Clean line art, using brand colors. Show workflow connections as node-and-line diagrams (matching n8n visual style). |
| **Screenshots** | Real n8n workflow screenshots, cropped cleanly, with subtle drop shadow on white/dark backgrounds. Annotate with coral red callout arrows when needed. |
| **Abstract graphics** | Subtle gradient meshes using coral → purple → teal. Used for hero backgrounds, section dividers. Never overwhelming. |
| **Social media** | Dark backgrounds (#0a0a0a) with coral/teal accent text. Screenshot of real work front and center. Minimal text overlay. |
| **Icons** | Lucide React icon set exclusively. 24px default, 1.5px stroke. Never mix icon sets. |
| **Photography** | No stock photo people. If human imagery is ever needed, use real photos of Anasan or the team. Candid, not staged. |
| **Forbidden** | No generic "AI brain" illustrations. No robot clipart. No handshake stock photos. No floating holographic UI mockups. |

### Animation Principles

- Duration: 150ms–300ms for micro-interactions, 400ms–600ms for page transitions
- Easing: `ease-out` for entrances, `ease-in` for exits, `spring` for interactive elements
- Purpose: Every animation must serve a function (guide attention, confirm action, show state change). No decorative animation.
- Reduced motion: Respect `prefers-reduced-motion` — provide static fallbacks

## 12. CASE STUDY FRAMEWORK

### How Results Should Be Presented

Every case study follows this exact format:

```
TITLE: [Client type] + [Core result]
Example: "Real Estate Agency Captures 3x More Leads with Zero Manual Follow-Up"

SECTIONS:

1. THE CLIENT (2-3 sentences)
   Who they are, size, industry, location.

2. THE PROBLEM (3-4 sentences)
   What was broken. How much time/money it was costing.
   Use specific numbers wherever possible.

3. WHAT WE BUILT (bullet list)
   Each workflow/automation delivered, described simply.
   Include tool names (n8n, Gmail, Slack, etc.)

4. THE RESULTS (numbered, with metrics)
   1. [Primary metric] — e.g., "12 hours/week saved"
   2. [Secondary metric] — e.g., "Lead response time: 45 min → 90 seconds"
```
